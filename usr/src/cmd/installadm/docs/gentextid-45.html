OpenSolaris 2010.03 Enhancements

  • You can do an automated installation of the OpenSolaris OS on SPARC or x86 client systems using media such a CD, DVD, or USB stick. You can install the client using either the default installation specifications provided in the media, or you can create and use a manifest with custom specifications. See Chapter 4, Automated Installations Using Media.

    Note - The client system must have network access. During the installation, the CD or other media will pull installation packages from an IPS repository on the network.

  • You can create and modify extended partitions as part of your automated installations. See Creating an Extended Partition.

  • You can install to an iSCSI disk target using the automated installer. See Using an iSCSI Target for Installations.

  • You can use the following expanded options for defining a target for an automated installation.

    • Boot disk

    • Device ID

    • Physical device path

    • Multiple criteria

    See Defining a Target for the Installation.

  • You can use the new installadm list command options to view information about your existing install services, associated install clients, and available manifests. See How to Review What You Have on Your Server.

  • The default behavior for booting SPARC and x86 systems over the network into AI images has changed as follows:

    • For SPARC systems, you must include the install argument in the boot command to initiate an installation as follows.

      For network installations:

      boot net:dhcp - install

      For installations using media:

      boot cdrom - install

      Note - If you use the command without the install argument, the install environment boots without initiating an installation.

    • For x86 network installations, you must select the second entry in the GRUB boot menu to initiate an automated installation. The first, default GRUB menu entry no longer starts a hands-free automated installation. Instead, the first entry boots into the AI boot image and presents a console-login, but automated installation does not start.

    • For x86 installations using media, if install=true is specified in the kernel line for the GRUB entry that you use, the installation automatically begins.

      Note - If you want to boot the x86 system without immediately starting an automated installation, check the GRUB menu entry that you're going to use. If install=true is specified in the kernel line for that GRUB entry, remove install=true. Then, the client will boot and a login screen will display, but the installation will not begin.
