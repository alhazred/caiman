OpenSolaris 2010.03 Enhancements

  • You can create an automated install (AI) CD, DVD, or, for x86 installations, a USB stick. Then, you can boot this AI media directly to initiate an automated installation. The system will still need network access, since the installation process retrieves packages from a network IPS repository. Booting the media directly is useful for cases where an AI server hasn't yet been set up on your network, or if you client system is unable to boot over the network to initiate a network AI install, for example, SPARC systems that do not have WAN Boot capability. See Chapter 4, Automated Installations That Boot From Media.

  • You can create and modify extended partitions as part of your automated installations. See Creating an Extended Partition (x86 Clients Only).

  • You can install to an iSCSI disk target using the automated installer. See Using an iSCSI Target for Installations.

  • You can use the following expanded options for defining a target for an automated installation.

    • Boot disk

    • Device ID

    • Physical device path

    • Multiple criteria

    See Defining a Target for the Installation.

  • You can use the new installadm list command options to view information about your existing install services, associated install clients, and available manifests. See How to Review What You Have on Your Server.

  • The default behavior for booting SPARC and x86 systems with AI images has changed as follows:

    • For SPARC systems, you must include the install argument in the boot command to initiate an installation as follows.

      For installations that boot over the network:

      boot net:dhcp - install

      For installations that boot from media:

      boot cdrom - install

      Note - If you use the command without the install argument, the install environment boots without initiating an installation.

    • For x86 installations that boot over the network, the GRUB menu displays as follows:

      OpenSolaris 2010.03 boot image
OpenSolaris 2010.03 Automated Install

      The default GRUB menu entry, “OpenSolaris 2010.03 boot image,” boots the image without starting a hands-free automated installation. You must select the second entry in the GRUB menu, “OpenSolaris 2010.03 Automated Install,” to initiate a “hands-free” automated installation.

    • For x86 installations that boot from media, the GRUB menu displays as follows:

      OpenSolaris 2010.03 Automated Install custom
OpenSolaris 2010.03 Automated Install
...
..

      The default GRUB menu entry, “OpenSolaris 2010.03 Automated Install custom,” prompts you for the location of a custom manifest before proceeding with the automated installation. You must select the second entry in the GRUB boot menu, “OpenSolaris 2010.03 Automated Install,” to initiate a “hands-free” automated installation. This selection uses the manifest on the media.

    • In general for x86 installations, if the install=true boot property is specified in the kernel line of the GRUB entry you are booting from, the installation automatically starts. If you want to boot your x86 based system without initiating an automated installation, check that the GRUB boot entry does not specify the install=true boot property. If the property is specified, edit the kernel line of boot entry and remove the property.

  • You can use the ISC DHCP server instead of the Solaris DHCP server. See 17. How do I use ISC DHCP With the Automated Installer?.
