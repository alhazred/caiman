Automated Install (Tasks)

Table 3-1 Task Map

Task

Question

Reference

Check hardware.

Do you have the necessary hardware for your install server and clients?

Any system that meets the requirements can be used as an automated install server or client, including laptops, desktops, virtual machines, and enterprise servers.

  • See the install server requirements at Server Setup.

  • See the client requirements at Client System Requirements.

Install the current OpenSolaris OS.

Is the current OpenSolaris OS installed on your install server?

You can use either a SPARC system or an x86 system as your install server.

You must have the current OpenSolaris release installed on the system that you plan to use as your install server. See Step 1 in How to Set Up an Install Server (Initial Steps).

Note - To set up a SPARC system as your install server, first create SPARC AI media, and then use that media to install the OpenSolaris release on that system. For instructions, see Chapter 4, Automated Installations That Boot From Media.

Set up a static IP address.

Do you have a static IP address set up for your install server?

If you do not have a static IP address set up for your install server, see Step 2 in Initial Setup.

Get the install server tools package.

Is the latest installadm package installed on your install server?

Note - Starting with the OpenSolaris 2010.03 release, the SUNWinstalladm-tools package has been renamed as the installadm package. Both the old and new names are usable.

For instructions about installing the latest installadm package, see Step 3 in Initial Setup.

Get an AI ISO image.

Do you have the AI ISO image you want to use as input for the installadm create-service command?

For instructions about downloading the AI ISO image, see Step 4 in Initial Setup.

Set up an install service.

Have you run installadm create-service command to set up your install service?

For instructions, see Chapter 5, Set up an Automated Install Server and Chapter 6, How to Administer an Automated Install Server.

Customize client installations.

Do you need to run the installadm create-client command to customize your client installations?

For instructions, see Chapter 7, How to Administer a Custom Install Client.

Replace default manifests.

Are the specifications in the default manifest suitable for your clients?

If the default manifest specifications are not suitable for your clients, you can create a custom manifest. See Chapter 8, How to Administer Manifest Files.

Assign custom manifests.

Do you want to augment the default manifest with a custom manifest?

If you want to assign custom manifests for use by specific clients, see How to Create and Use a Criteria Manifest.

Start installations.

Are you ready to start client installations?

Boot the client from the network. For x86 clients, select the network device to boot from. For SPARC clients, at the OK open boot prompt, type boot net:dhcp - install. See Chapter 9, How to Manage Client Installations.

Troubleshooting

Was there a problem with your installation?

Refer to Chapter 10, Troubleshooting and FAQs for the Automated Installer. See, also, Chapter 9, How to Manage Client Installations.
